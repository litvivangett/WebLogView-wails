# Copilot Context - WailsLogView Project

**Last Updated:** May 7, 2026

## Project Overview
WailsLogView is a native cross-platform desktop log viewer built with Wails v3, featuring real-time file monitoring and Kubernetes pod log streaming. It runs on Windows, macOS, and Linux as a standalone desktop application.

## Tech Stack
- **Backend:** Go 1.25+
  - Wails v3 (v3.0.0-alpha.80) for desktop application framework
  - fsnotify for file system monitoring
  - k8s.io/client-go for Kubernetes integration
- **Frontend:** Preact (with virtual scrolling via react-window)
  - @wailsio/runtime for Wails event system and bindings
  - ansi-to-html for terminal color rendering
  - Vite build system
- **Communication:** Wails IPC bindings (direct Go function calls) + Wails event system (real-time streaming)

## Key Features
- **Native desktop app** (Wails v3 - no browser required)
- Real-time log file monitoring with fsnotify
- Kubernetes pod log streaming (multi-cluster, namespace/pod/container discovery)
- Virtual scrolling for large files (react-window)
- Regex filtering (include/exclude patterns with OR/AND logic)
- Tabbed interface (Chrome-like multi-file support)
- Dual-pane layout (all lines + filtered lines)
- Resizable panes with draggable divider
- Horizontal scrolling for long log lines
- **Click-to-highlight navigation** - Click line number in filtered pane to jump to that line in all lines pane
- **ANSI color rendering** - Terminal colors displayed properly (using ansi-to-html)
- **File drag & drop** - Drop log files directly onto the window
- **Persistent settings** stored in `~/.weblogview/settings.json`
- Configurable initial window size (tail lines)
- Configurable ANSI rendering per pane (prettify or show raw)
- Cross-platform (Windows/macOS/Linux)
- Single executable with embedded frontend (no dependencies)

## Architecture - Wails Services
The backend exposes functionality to the frontend via Wails services (registered in `main.go`):
- **FileService** (`internal/handlers/file/`) - Open files for log streaming
- **K8sService** (`internal/handlers/k8s/`) - Kubernetes operations (contexts, namespaces, pods, containers, log streaming)
- **SettingsService** (`internal/handlers/settings/`) - Get/update application settings
- **RecentService** (`internal/handlers/recent/`) - Recently opened files and namespaces
- **SessionService** (`internal/session/`) - Session lifecycle management (close sessions)

## Communication Pattern
- **Function calls:** Frontend calls Go functions directly via auto-generated Wails bindings (`frontend/bindings/`)
- **Real-time streaming:** Backend emits events via Wails event system:
  - `log-initial` - Initial batch of log lines when opening a source
  - `log-lines` - New log lines as they arrive
  - `log-error` - Error notifications
  - `file-dropped` - File drag & drop events

## Settings System
- **Backend:** `/internal/settings/settings.go`
  - Singleton pattern with thread-safe operations
  - Persists to `~/.weblogview/settings.json`
  - Cross-platform compatible (uses `os.UserHomeDir()` and `filepath.Join()`)
- **Frontend:** Settings modal (⚙️ button in control bar)
  - Initial Window Size: 100-100,000 lines (default: 1000)
  - ANSI Rendering for Top Pane: on/off (default: on - prettified)
  - ANSI Rendering for Bottom Pane: on/off (default: on - prettified)
  - Settings saved on backend and persist across restarts
- **ANSI Rendering:** Uses `ansi-to-html` library to convert terminal color codes to HTML
  - Configurable per pane (top/bottom)
  - When enabled: Shows colored logs like in terminal
  - When disabled: Shows raw ANSI escape codes

## Development Commands
```bash
# Install dependencies
go mod download
cd frontend && npm install && cd ..

# Run in development mode (hot reload for both frontend and backend)
wails3 dev

# Production build
wails3 build

# Generate TypeScript bindings (after changing Go services)
wails3 generate bindings -ts

# Run with task runner
task dev
task build
```

## Build Output
Production builds create standalone executables in `build/`:
- `WailsLogView-darwin-arm64` (macOS Apple Silicon)
- `WailsLogView-darwin-amd64` (macOS Intel)
- `WailsLogView-linux-amd64`
- `WailsLogView-windows-amd64.exe`

Each executable includes the embedded frontend - no external dependencies required.

## Notes
- Project uses Go embed directive (`//go:embed all:frontend/dist`) to include frontend in binary
- Frontend is built by Vite to `frontend/dist/` which is embedded at compile time
- Development mode: `wails3 dev` runs both frontend and backend with hot reload
- Production mode: Single native executable with embedded webview
- Settings file location: `~/.weblogview/settings.json` (all platforms)
- Wails bindings are auto-generated - never write them manually
- Go dependencies are vendored in `vendor/`
